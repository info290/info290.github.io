Info 290T: Human-in-the-loop Data Mgmt.

This class will be a modern take on a traditional data management class, covering the basics of dealing with data at scale from a user-centered perspective, over the entire life cycle of data management, ranging from data cleaning, extraction, and integration, to analysis and exploration, to machine learning and collaboration.

Description

The class will not assume any background in databases, and will start with a quick overview of the classics: the relational model, SQL, views and indexes, query processing and optimization, and transactions, all from a user perspective.

Then, the class will cover a range of modern or non-traditional data systems. We will cover some subset of: graph processing systems, dataframe systems, map-reduce and other distributed processing systems, spreadsheet systems, information retrieval and search systems, document stores, column stores, and stream processing systems. We will also cover techniques or aspects essential to modern data systems, such as: usability concerns, compression and data encoding formats, data cleaning and integration, approximation and sketching, summarization (OLAP and visualization), and managing security and privacy.

The class will mix traditional lectures and assignments (both written and programming-oriented), along with a class project, which could be research or programming-oriented. Experience with programming, a basic understanding of computer systems, data structures, and algorithms expected.

Since this is the first time this class is being offered, everything, including the breakdown of topics and grading is tentative; please expect hiccups if you're taking the class, and apologies for any issues in advance!

Relationship to INFO257

This class is meant to be roughly equivalent to INFO257 (Database Systems), and if you are a MIMS student, you are free to pick either one to fulfill requirements. That class is a more standard, polished take on database systems, while this class is a more modern take, emphasizing user-centered aspects (but beware of hiccups that come with a new class!)

Reading/Textbook

Since this class is a modern take on data management, emphasizing a user perspective, there isn't a good textbook that covers all the topics in the class. The topics covered early on in the class, i.e., "the classics", are adequately handled in most database textbooks. One appropriate resource is "Database Systems: The Complete Book" by Garcia-Molina, Ullman, and Widom. The modern topics are typically not adequately covered in database textbooks. We will provide supplementary readings if there is interest.

Breakdown (tentative)

The class will involve five assignments (three written and two programming assignments), plus a semester-long project.

  • Written Assignments (30%): There will be three written assignments of 10% each, typically due on Friday midnight. Each written assignment is meant to ensure understanding of roughly 1/3rd of the class content, and will contain a list of questions. Responses should be written in a way that succinctly describes the solution or concept.
  • Programming Assignments (30%): There will be two programming assignments of 15% each, typically due Friday midnight. The programming assignments are meant to provide hands-on experience working on classical and modern data systems. The programming assignments is meant to be done in teams of two, but please talk to us in case of exceptions.
  • Project (35%): The project is meant to test application of class material in new settings, or extending the class materials in various ways. The project will either involve research, or development, or both. We will provide feedback regarding what would be an acceptable project topic as the class proceeds. The project is meant to be done in teams of two, but please talk to us in case of exceptions.
    • Midterm report (5%, due 3/30 midnight)
    • Final report + presentation (30%, report due: 05/01 midnight; presentation slides due: 4/27 prior to class)
  • Class participation (5%): To incentivize participation in class, 5% of the class grade will be reserved for class participation.

Late Policy: Students are allowed to use up to four late days, to be used in any way that makes sense, without any explanation necessary. Any submission after the deadline will be rounded up to the closest day (i.e., 24 hour window) -- so even one hour late will count as the use of a full day. After four late days, students will lose 1% of the grade per day late. In case of medical issues, please contact the instructors as soon as you are able.

Project

As part of this class, you need to complete a semester-long project. The details of the project will be announced later. You are free to look for ideas in your domain of expertise: for instance, if you work in computational journalism, building a new way to browse and manage large collections of textual archives could be a perfectly reasonable project. Either way, you must speak to the instructor to verify that the project is indeed "challenging" enough.
